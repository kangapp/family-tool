# 家庭电费工具微信小程序第一版需求方案

## 目标

做一个家庭电费分摊小工具。用户每月在小程序里录入各楼层电表读数，系统按内置 Excel 模板规则自动计算费用，并生成一张适合微信群转发的电费明细截图。

第一版不做上传 Excel、照片 OCR、登录、多家庭、云同步。Excel 文件只作为内置模板和公式参考。

## 用户流程

1. 打开小程序进入电费明细编辑页。
2. 选择或填写月份。
3. 填写总表电费、电费每度。
4. 在表格里编辑各住户的上月底数、本月读数、是否平摊、管理费。
5. 页面实时计算本月用电量、电费、分摊费、合计。
6. 点击生成图片。
7. 保存图片或转发到微信群。

## 页面结构

第一版只做一个页面。

- 基础信息区：月份、总表电费、电费每度。
- 明细编辑区：参考纸质电费明细表，表格行固定。
- 操作区：重置、生成图片、保存图片。
- 预览区：显示生成后的明细图。

固定住户：

- 一楼101
- 一楼102
- 二楼
- 三楼
- 四楼
- 五楼
- 六楼
- 七楼
- 八楼

## 字段

每行字段：

- 楼层住户：固定文本。
- 上月底数：用户输入。
- 本月读数：用户输入。
- 本月用电量：自动计算。
- 电费：自动计算。
- 是否平摊：用户选择。
- 分摊费：自动计算。
- 管理费：默认生成，允许用户改。
- 合计：自动计算。

基础字段：

- 月份。
- 总表电费。
- 电费每度。

## 计算规则

本月用电量：

```text
本月读数 - 上月底数
```

电费：

```text
本月用电量 × 电费每度
```

剩余待分摊金额：

```text
总表电费 - 各户电费合计
```

分摊费：

```text
剩余待分摊金额 ÷ 是否平摊为“是”的户数
```

合计：

```text
电费 + 分摊费 + 管理费
```

管理费默认值：

- 一楼101、一楼102：有用电量时默认 10 元，否则 0 元。
- 二楼到八楼：有用电量时默认 20 元，否则 0 元。

金额显示保留 2 位小数。用电量显示整数。

## 校验

第一版只做必要校验。

- 本月读数不能小于上月底数。
- 总表电费、电费每度不能小于 0。
- 参与平摊户数为 0 时，分摊费显示 0，并提示无法分摊。
- 输入为空时按 0 参与计算，但输入框保留为空，方便继续编辑。

## 图片生成

生成图使用微信小程序 Canvas 绘制。

样式参考纸质模板：

- 标题：`YYYY年MM月电费明细表`
- 表头：楼层住户、上月底数、本月读数、本月用电量、电费、分摊费、管理费、合计。
- 表格线清晰，适合手机查看和微信群传播。
- 底部显示总表电费、电费每度、生成时间。

## 数据保存

第一版使用本地缓存保存当前编辑内容。

- 用户退出后再次打开，恢复上次数据。
- 提供重置按钮，恢复内置模板默认值。

## 微信小程序开发流程

1. 注册小程序账号，获取 AppID。
2. 安装微信开发者工具。
3. 新建小程序项目，选择原生小程序。
4. 搭建页面：
   - `pages/index`：电费编辑、计算、图片生成。
   - `utils/calc`：电费计算函数。
   - `utils/default-data`：内置住户和默认值。
5. 实现在线编辑表格。
6. 实现计算逻辑。
7. 实现本地缓存。
8. 实现 Canvas 图片生成和保存。
9. 使用微信开发者工具真机预览。
10. 提交体验版，手机完整走一遍录入、生成、保存流程。
11. 提交审核。
12. 发布正式版。

## 验收标准

- 能编辑月份、总表电费、电费每度。
- 能编辑 9 个固定住户的读数、是否平摊、管理费。
- 修改任一输入后，费用实时更新。
- 计算结果与内置 Excel 模板规则一致。
- 能生成清晰明细图片。
- 图片能保存到相册。
- 关闭小程序后再打开，未重置前数据仍在。

## 暂不做

- 上传 Excel。
- 识别照片或手写数字。
- 多套模板。
- 多家庭/多房屋。
- 云端同步。
- 后台管理。
- 登录注册。
